# QA Standard – Test Case Structure

## Required Fields
- Test Case ID
- Title
- Description
- Preconditions
- Test Steps
- Expected Result
- Postconditions
- Test Data
- Priority (High/Medium/Low)
- Related Requirement / Story ID
- Tags (API, Regression, Smoke, Integration…)

## Conventions
- Steps must be written in numbered format.
- Expected results must be measurable and objective.
- Negative cases require explicit validation messages.
- Must include at least 3 edge cases whenever applicable.

## Style
- Clear, concise, professional.
- No assumptions—explicitly state all data.
- Use active voice (e.g., "User clicks", "System returns").
